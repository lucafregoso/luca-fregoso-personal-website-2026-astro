# Welcome to `ai-engineering`

Thanks for installing ai-engineering. This is your post-install manual: a
simple, complete reference to everything the framework can do, so you always
know what each capability is for and can reach the right one fast.

ai-engineering installs a governance layer into this repository as ordinary
versioned files — specs, plans, skills, agents, runbooks, hooks, and an audit
trail. Nothing is hidden in a hosted control plane. You drive it from your IDE
agent surface with `/ai-<name>` skills.

`54 skills · 9 agents · 6 surfaces · 1 governed flow`

## Quick wins

The fastest path to value. Start here, then move through the governed chain.

```text
/ai-start
```

- `/ai-start` — orient in any repository: it reads the project, reports state,
  and tells you the safe next command. Run it first in a new session.

The canonical delivery chain takes work from idea to merged PR with governance
at every step. Reach for it whenever a change touches product behavior,
framework behavior, security posture, public docs, or release state.

```text
/ai-brainstorm → /ai-plan → /ai-build → /ai-pr
```

- `/ai-brainstorm` — interrogate the design and produce an approved spec. Run
  it before writing code for anything non-trivial; the spec becomes the
  contract for everything downstream.
- `/ai-plan` — turn the approved spec into a phased, patch-ready plan and pick
  the executor route (`/ai-build` or `/ai-autopilot`).
- `/ai-build` — execute the plan: stack-aware implementation, TDD
  self-validation per task, and a final quality loop before the PR.
- `/ai-pr` — run the verify + review + commit pipeline and open the pull
  request, watching CI until it is green.

`/ai-commit` stays available for work-in-progress checkpoints; it is not part of
the canonical chain.

## Maintainer reference

This manual is the operator surface. The framework's internal contracts live in
their own canonical homes — read these when you need the plumbing, not the
day-to-day usage:

- [docs/persistence-doctrine.md](../docs/persistence-doctrine.md) — where every
  datum lives (the canonical three-tier, files-only persistence model) and the
  rebuild semantics for derived caches.
- [CONSTITUTION.md](../CONSTITUTION.md) — project identity, ownership
  boundaries, and the hard prohibitions every change must respect.
- [manifest.yml](manifest.yml) — the configuration source of truth (stacks,
  surfaces, retention, telemetry consent).

Update canonical sources, then regenerate the derived surfaces:

```bash
ai-eng dev sync
ai-eng dev sync --check
```
